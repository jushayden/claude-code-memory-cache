# Claude Code Cache Engine

Two guides for giving **Claude Code** a persistent, token-efficient memory that survives across sessions, projects, and machines — a "cache" for everything the agent should remember between runs.

- **[CLAUDE_SECOND_BRAIN.md](CLAUDE_SECOND_BRAIN.md)** — the full system: the 5 memory layers, the memory-server code (inline, copy-pasteable), and setup (manual + "let your Claude install it").
- **[TOKEN_EFFICIENCY.md](TOKEN_EFFICIENCY.md)** — a focused deep-dive on cutting token usage (the 7 techniques + the tools/config that enable them).

Start with `CLAUDE_SECOND_BRAIN.md` if you want the whole system; read `TOKEN_EFFICIENCY.md` if you just want the token savings.
